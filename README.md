# pr-farm

A tiny sandbox repository for harmless GitHub workflow and contribution-graph experiments.

## Purpose

- Test basic Git/GitHub flows without touching real projects.
- Keep achievement/contribution experiments isolated from production repositories.
- Provide a low-risk place for automation smoke tests.

## Rules

- No secrets or credentials.
- No generated spam across other repositories.
- No dependency on external services.
- Keep commits small, reversible, and intentionally boring.

## Files

- `counter.txt` — simple mutable state for sandbox commits.
